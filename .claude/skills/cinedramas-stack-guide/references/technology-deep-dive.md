# Technology Deep Dive

Source: Master Architecture Blueprint, Section 3

For each technology: what it is, how it works, why chosen, how it integrates.

## React Native
- Native UI from JS/TS via Hermes engine. Bridge/JSI/Fabric translates React tree to native components.
- Single codebase -> iOS + Android. Flexbox via Yoga layout engine.
- All screens, components, hooks in TypeScript. Platform-specific via `Platform.OS` checks.

## Expo + EAS
- Managed workflow: native projects generated by Expo. Custom dev client for extra native modules.
- EAS Build: cloud compilation with signed binaries. EAS Update: OTA JS bundle updates per release channel.
- `app.config.js` reads `APP_VARIANT` -> loads `brands/{variant}/manifest.json`. New white-label = new brand dir + build profile.

## Expo Router
- File-system routing in `app/` directory. Wraps React Navigation.
- `app/(tabs)/_layout.tsx` = tab navigator. `app/series/[id].tsx` = dynamic route.
- Auto deep linking: each file route = URL path.

## react-native-video v7
- iOS: AVPlayer. Android: ExoPlayer. Native HLS adaptive bitrate support.
- v7 `VideoPlayer.preload()` enables pre-buffering next/prev episodes.
- Wrapped in `components/video/VideoPlayer.tsx`. PreloadManager manages pool of N-1/N+1.

## @shopify/flash-list v2
- Cell recycling (reuses rendered components, changes data only). Up to 5x faster than FlatList.
- Requires `estimatedItemSize`. Used horizontally (home rails) and vertically (video feed with paging + snap).

## NativeWind
- Tailwind CSS -> RN StyleSheet via Babel plugin at build time.
- CSS custom properties enable runtime theming: `--color-primary` swapped per brand.
- `global.css` defines defaults. Brand `theme.ts` overrides. ThemeProvider applies.

## Zustand
- `create()` returns a hook. Selector-based subscriptions: `useAuthStore(s => s.user)`.
- Three stores: authStore (user, tenant_id, JWT), playerStore (episode, position), configStore (theme, features).
- No providers, no reducers. Prevents feed re-renders (critical for 60 FPS).

## TanStack React Query
- Query key -> cached response. staleTime controls freshness. Background refetch.
- `persistQueryClient` serializes to AsyncStorage -> offline-first cold start < 2s.
- Hooks: useCatalog (5min stale), useWatchProgress (30s stale), useEntitlements.

## Supabase
- PostgreSQL + PostgREST (auto REST API) + RLS + Auth (GoTrue) + Edge Functions (Deno) + Storage (S3-compatible).
- PgBouncer transaction-mode pooling for serverless workloads.
- Silhouette: dedicated project per tenant. Hub: shared project, RLS scopes by tenant_id.

## Mux
- Ingest -> encode (360p-4K) -> HLS delivery (segments + m3u8 manifest) -> adaptive bitrate.
- Playback IDs: public (open) or signed (RS256 JWT with sub=playbackId, aud="v", exp, kid).
- Mux Data: startup time, rebuffering, engagement. SDK: `@mux/mux-data-react-native-video`.
- Webhooks: `video.asset.ready`, `video.asset.errored`. Verify via `Mux-Signature` HMAC-SHA256.

## RevenueCat
- `react-native-purchases` SDK. Init with public API key. Offerings fetched at runtime.
- Entitlements: products mapped to entitlements (e.g., Monthly + Yearly -> "premium").
- Receipt validation server-side by RevenueCat. Webhooks: initial_purchase, renewal, cancellation, expiration.
- Per-tenant project with tenant-specific API keys.

## Cloudflare
- 300+ PoPs. DDoS + WAF -> cache check -> origin if miss.
- Must explicitly configure JSON caching via Cache Rules. Default: JSON NOT cached.
- `stale-while-revalidate`: serve stale immediately, refresh in background.
- Cache key includes tenant_id to prevent cross-tenant pollution.

## Sentry
- `@sentry/react-native`: JS exceptions, native crashes, performance traces.
- Source maps: auto-uploaded via Sentry Expo plugin during EAS Build.
- Tags: tenant_id set after login. Error boundaries per screen. Navigation breadcrumbs.

## Supporting Libraries
- **react-native-reanimated**: UI-thread worklet animations. 60 FPS gesture-driven.
- **react-native-gesture-handler**: Native-thread gesture recognition. Composable gestures.
- **expo-secure-store**: Encrypted Keychain/EncryptedSharedPreferences for JWTs.
- **expo-image**: Disk caching, progressive loading, blurhash placeholders for Mux thumbnails.
